# Rails 8 + Docker Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up a Rails 8 application with Docker Compose for PostgreSQL (pgvector) and Redis, AlpineJS via importmaps, and Bootstrap via cssbundling-rails.

**Architecture:** Docker Compose runs Postgres and Redis as services; Rails runs on the host for fast iteration. The existing Playwright E2E test setup is preserved alongside the new Rails app.

**Tech Stack:** Ruby 3.4, Rails 8, PostgreSQL 17 (pgvector), Redis 7, AlpineJS, Bootstrap, importmaps

**Spec:** `docs/superpowers/specs/2026-03-22-rails-docker-setup-design.md`

### Task 1: Docker Compose setup

**Files:**
- Create: `docker-compose.yml`
- Create: `.env.example`

- [ ] **Step 1: Create `docker-compose.yml`**

```yaml
services:
  postgres:
    image: pgvector/pgvector:pg17
    ports:
      - "5432:5432"
    environment:
      POSTGRES_USER: ${POSTGRES_USER:-financebuddy}
      POSTGRES_PASSWORD: ${POSTGRES_PASSWORD:-password}
      POSTGRES_DB: ${POSTGRES_DB:-financebuddy_development}
    volumes:
      - financebuddy_postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U financebuddy"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"
    volumes:
      - financebuddy_redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 5s
      retries: 5

volumes:
  financebuddy_postgres_data:
  financebuddy_redis_data:
```

- [ ] **Step 2: Create `.env.example`**

```
POSTGRES_USER=financebuddy
POSTGRES_PASSWORD=password
POSTGRES_DB=financebuddy_development
DATABASE_HOST=localhost
DATABASE_PORT=5432
DATABASE_USERNAME=financebuddy
DATABASE_PASSWORD=password
REDIS_URL=redis://localhost:6379/0
```

- [ ] **Step 3: Copy `.env.example` to `.env`**

Run: `cp .env.example .env`

- [ ] **Step 4: Stop any local Postgres/Redis and start Docker services**

If a local Postgres or Redis is running on ports 5432/6379, stop it first to avoid port conflicts:

```bash
sudo systemctl stop postgresql 2>/dev/null; sudo systemctl stop redis 2>/dev/null; true
```

Run: `docker compose up -d`

Wait for health checks to pass (up to 30 seconds):

Run: `docker compose up --wait`

Run: `docker compose ps`

Expected: Both `postgres` and `redis` show status "healthy".

Run: `docker compose exec postgres psql -U financebuddy -c "SELECT 1;"`

Expected: Returns a row with `1`.

Run: `docker compose exec redis redis-cli ping`

Expected: `PONG`

- [ ] **Step 5: Verify pgvector extension is available**

Run: `docker compose exec postgres psql -U financebuddy -d financebuddy_development -c "CREATE EXTENSION IF NOT EXISTS vector; SELECT extname FROM pg_extension WHERE extname = 'vector';"`

Expected: Output includes `vector` in the result.

- [ ] **Step 6: Commit**

```bash
git add docker-compose.yml .env.example
git commit -m "Add Docker Compose for PostgreSQL (pgvector) and Redis"
```

### Task 6: Enable pgvector extension

**Files:**
- Create: `db/migrate/XXXXXX_enable_pgvector.rb` (via Rails generator)

- [ ] **Step 1: Generate the migration**

Run: `bin/rails generate migration EnablePgvector`

- [ ] **Step 2: Edit the migration file**

Open the generated file at `db/migrate/*_enable_pgvector.rb` and set its contents to:

```ruby
class EnablePgvector < ActiveRecord::Migration[8.0]
  def change
    enable_extension 'vector'
  end
end
```

- [ ] **Step 3: Run the migration**

Run: `bin/rails db:migrate`

Expected: Completes successfully.

- [ ] **Step 4: Verify the extension is enabled**

Run: `bin/rails runner "puts ActiveRecord::Base.connection.execute(\"SELECT extname FROM pg_extension WHERE extname = 'vector'\").first"`

Expected: Output includes `vector`.

- [ ] **Step 5: Commit**

```bash
git add db/migrate/*_enable_pgvector.rb db/schema.rb
git commit -m "Enable pgvector extension via migration"
```
