# Data Model Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement all 15 database tables defined in the data model design spec, with Rails models, validations, associations, and model-level tests.

**Architecture:** Rails migrations create the tables with proper indexes and foreign keys. Each model has validations and associations matching the spec. Tests use Rails fixtures and Minitest (the project default). The Rails 8 authentication generator handles `users` and `sessions`. All other tables are created via hand-written migrations.

**Tech Stack:** Ruby on Rails 8, PostgreSQL 17, Minitest, Rails fixtures

**Spec:** `docs/superpowers/specs/2026-03-22-datamodel-design.md`

**Branch:** `feature/data-model`

**Sign conventions for double-entry:**
- Asset accounts (cash, investment): positive balance = money owned
- Liability accounts (credit, loan): negative balance = money owed
- Expense accounts: positive balance = money spent
- Revenue accounts: negative balance = money earned
- Transaction lines: positive = debit, negative = credit, must sum to zero per transaction
